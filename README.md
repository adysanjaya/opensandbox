# Sandbox - Flow Builder for API Endpoints

A monorepo project that lets users build dynamic HTTP endpoints using a visual flow builder (React Flow). Inspired by the flow builder concept in waengine, but adapted for standard HTTP responses instead of WhatsApp messaging.

**🚀 Live Demo:** [https://sandbox.getpulsa.co.id/](https://sandbox.getpulsa.co.id/)

## Architecture

```
sandbox/
├── apps/
│   ├── api/          # Hono + Drizzle ORM + PostgreSQL + Flow Engine
│   └── web/          # Next.js + React Flow + Tailwind CSS
└── packages/
    └── types/        # Shared TypeScript types
```

## Features

- **User Authentication**: JWT-based auth with register/login
- **Dynamic Endpoints**: Each user gets a unique hash-based URL
  - Format: `/{userHash}/{endpointSlug}`
- **Visual Flow Builder**: Drag-and-drop nodes powered by React Flow
- **Flow Nodes**:
  - **Trigger**: Entry point with HTTP method selection
  - **Condition**: Branching logic (equals, contains, startsWith, endsWith, regex)
  - **HTTP Request**: Call external APIs with variable substitution
  - **Set Variable**: Define variables using templates
  - **Transform**: JSON path extraction, template rendering, uppercase/lowercase, parse/stringify
  - **Delay**: Wait for a specified duration
  - **A/B Split**: Random branching with configurable percentages
  - **Response**: Final HTTP response (JSON, Text, or XML) with dynamic body and headers
- **Variable System**: Use `{{variables.name}}`, `{{query.param}}`, `{{body}}`, `{{headers.name}}` anywhere
- **Response Types**: Text, JSON, XML with customizable status codes and headers

## Database Migration (Safe)

This project uses proper SQL migration files generated by Drizzle Kit:

```bash
# Generate migration SQL from schema changes
bun run db:generate

# Apply migrations to database
bun run db:migrate

# Open Drizzle Studio
bun run db:studio
```

Migrations are tracked in the `drizzle_migrations` table to prevent double-running.

## Quick Start

### Prerequisites
- Bun 1.2+
- PostgreSQL running (default: `waengine-postgres-1` Docker container on port 5433)
- PM2 installed globally (`npm i -g pm2`)

### Install Dependencies
```bash
cd sandbox
bun install
```

### Setup Database
Make sure database `sandbox` exists in your PostgreSQL instance, then:
```bash
bun run db:generate
bun run db:migrate
```

### Environment Variables

Create `apps/api/.env`:
```env
DATABASE_URL=postgres://weaku:devpassword123@localhost:5433/sandbox
JWT_SECRET=your-super-secret-key
PORT=4000
```

Create `apps/web/.env.local`:
```env
NEXT_PUBLIC_API_URL=http://localhost:4000
API_URL=http://localhost:4000
```

## Running with PM2

### Option A: Development Mode (manual, recommended for coding)
```bash
# Terminal 1 - API
bun run dev:api

# Terminal 2 - Web
bun run dev:web
```

### Option B: Development Mode via PM2
```bash
# Build API first
bun run build:api

# Start both services via PM2
bun run pm2:start

# Check status
bun run pm2:status

# View logs
bun run pm2:logs

# Stop all
bun run pm2:stop

# Restart all
bun run pm2:restart
```

### Option C: Production Mode via PM2
```bash
# Build everything
bun run build

# Start in production mode
bun run pm2:start:prod

# Save PM2 config to auto-restart on boot
bun run pm2:save
pm2 startup
```

## API Endpoints

### Auth
- `POST /api/auth/register` - Register new user
- `POST /api/auth/login` - Login
- `GET /api/auth/me` - Get current user

### Endpoints (requires Bearer token)
- `GET /api/endpoints` - List my endpoints
- `POST /api/endpoints` - Create endpoint
- `GET /api/endpoints/:id` - Get endpoint
- `PATCH /api/endpoints/:id` - Update endpoint
- `DELETE /api/endpoints/:id` - Delete endpoint

### Public Execution
- `ANY /:userHash/:slug` - Execute endpoint flow

## Flow Engine Execution

When a request hits `/{userHash}/{slug}`:
1. User and endpoint are resolved from PostgreSQL
2. Flow JSON is loaded and parsed
3. Execution context is built from the HTTP request
4. Flow engine processes nodes starting from Trigger
5. Variables are substituted using `{{...}}` syntax
6. Response node produces the final HTTP response
